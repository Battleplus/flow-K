# FLOW AI Trader · 三维分析架构 v2

> 消息面定方向 → 板块ETF确认资金 → 个股K线择时进场

## 1. 核心理念

### 1.1 为什么要做三维，而不是纯K线

纯K线统计解决了"什么时候买"的问题，但没解决"买什么"的问题：

- 只看个股K线：信号触发了，但大盘/板块在跌，胜率大打折扣
- 加上板块ETF：先确认板块资金在流入，个股信号的可信度更高
- 再加上消息面：知道当前什么题材在发酵，能主动选对赛道

### 1.2 资金轮动逻辑（核心洞察）

**美股市场的资金总量是守恒的——资金不会凭空消失，只会在板块之间轮动。**

不需要看复杂的Level2资金流数据，板块ETF本身就是资金流向的"水表"：
- 一个板块ETF放量上涨 = 资金在流入这个板块
- 一个板块ETF放量下跌 = 资金从这个板块流出
- 11个板块的相对强度排名，直接告诉你今天钱在往哪走、从哪撤

一句话总结：**先选赛道（消息面+板块ETF），再选选手（个股K线信号）。**

## 3. 第一层：消息面分析（News Layer）

### 3.1 模块位置
`src/news_analysis.py`（新增）

### 3.2 输入输出
- **输入**：一段自然语言新闻/事件文本（如"美联储宣布降息50基点，AI芯片需求超预期"）
- **输出**：11个板块各自的消息面评分，范围 `-2 ~ +2`

### 3.3 处理流程

```
用户输入文本
    ↓
┌─────────────────────────────────┐
│ 1. 关键词匹配 (权重 0.4)         │
│    查 event_sector_mapping.json  │
│    命中关键词 → 对应板块+分       │
│    情感词(如"超预期""禁令")加权  │
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│ 2. TF-IDF语义匹配 (权重 0.6)     │
│    文本向量化                     │
│    与470只股票BusinessSummary    │
│    计算余弦相似度                 │
│    按板块聚合取TopN平均分         │
└─────────────────────────────────┘
    ↓
融合评分 → 11板块 news_score[-2, +2]
```

### 3.4 评分规则

| 评分 | 含义 | 触发条件 |
|------|------|----------|
| +2 | 强利好 | 直接命中核心关键词 + TF-IDF语义高分 |
| +1 | 弱利好 | 间接相关或单一信号 |
| 0 | 中性 | 无相关/多空对冲 |
| -1 | 弱利空 | 负面关键词命中 |
| -2 | 强利空 | 核心负面事件（制裁、禁令、暴雷） |

### 3.5 依赖数据
- `data/event_sector_mapping.json`：已有的17组关键词→板块映射
- `data/company_info.csv`：503家公司的BusinessSummary，已有TF-IDF向量库（470股×2000维）

### 3.6 设计亮点
- **不需要实时新闻API**：新闻由用户/评委现场输入，系统做分析判断
- **关键词+语义双保险**：既有精确规则匹配，又有TF-IDF覆盖未在词表中的新事件
- **解释性强**：可以告诉用户"因为你说的新闻里提到了'AI芯片'，所以命中科技板块，同时语义相似度0.87"

## 4. 第二层：板块ETF + 资金轮动（Sector Rotation Layer）

### 4.1 模块位置
`src/sector_rotation.py`（新增）

### 4.2 输入输出
- **输入**：板块ETF历史行情数据（sector_etf_prices.csv）
- **输出**：
  - 11板块的趋势评分（-1~+1）
  - 板块相对强度排名RS（近20/60日超额收益）
  - 资金流入/流出判断
  - 板块综合评分（融合消息面）

### 4.3 SPDR 11板块ETF清单

| ETF | 板块中文名 | 英文板块名 |
|-----|-----------|-----------|
| XLK | 科技 | Technology |
| XLV | 医疗 | Health Care |
| XLF | 金融 | Financials |
| XLE | 能源 | Energy |
| XLI | 工业 | Industrials |
| XLY | 可选消费 | Consumer Discretionary |
| XLP | 必需消费 | Consumer Staples |
| XLU | 公用事业 | Utilities |
| XLB | 材料 | Materials |
| XLRE | 房地产 | Real Estate |
| XLC | 通信服务 | Communication Services |
| SPY | 基准 | S&P 500（基准对照） |

### 4.4 三个子评分

#### (1) ETF趋势评分 `etf_trend_score ∈ {-1, 0, +1}`
**完全复用现有代码！**对每只ETF直接调用：
- `patterns.py` 的 `detect_all()` → 35个K线信号
- `strategies.py` 的 `aggregate_signals()` → 多策略共识
- 判断标准：
  - MA多头排列 + MACD金叉 + 斜率向上 → +1（上升趋势）
  - MA交织 / 信号矛盾 → 0（横盘）
  - MA空头排列 + MACD死叉 + 斜率向下 → -1（下降趋势）
- 额外加分：放量上涨+0.3，放量下跌-0.3（满分钳制在±1）

#### (2) 相对强度评分 `rs_score ∈ {-1, 0, +1}`
核心是**和SPY比超额收益**：
```
对每只ETF计算:
  return_20d = (当前价 / 20日前价) - 1
  return_60d = (当前价 / 60日前价) - 1
  excess_20d = return_20d - SPY同期收益
  excess_60d = return_60d - SPY同期收益
  rs_raw = 0.6 * z-score(excess_20d) + 0.4 * z-score(excess_60d)

按rs_raw排名1~11:
  排名1-3 → +1 (强势区，资金流入)
  排名4-8 →  0 (中性区)
  排名9-11→ -1 (弱势区，资金流出)
```

#### (3) 资金轮动确认矩阵
| ETF趋势 | 相对强度 | 资金判断 | 操作建议 |
|---------|---------|---------|---------|
| 上升趋势 (+1) | 强势区 (+1) | ✅ **资金确认流入** | 最强信号，重点选股 |
| 上升趋势 (+1) | 弱势区 (-1) | ⚠️ 高位派发 | 警惕见顶，不加仓 |
| 下降趋势 (-1) | 强势区 (+1) | 🔄 板块轮动早期 | 弱势转强，重点关注 |
| 下降趋势 (-1) | 弱势区 (-1) | ❌ **资金确认流出** | 坚决回避 |
| 横盘 (0) | 任意 | ⏸️ 无明确方向 | 观望 |

### 4.5 板块综合评分

```
sector_score = news_score * 0.4       # 消息面权重40%
             + etf_trend_score * 0.3  # ETF趋势权重30%
             + rs_score * 0.3         # 相对强度权重30%
```

取值范围 `-2.0 ~ +2.0`

### 4.6 板块分级
- **🔥 强势板块**：sector_score ≥ +0.8 → 在这里选股
- **👀 关注板块**：+0.3 ≤ sector_score < +0.8 → 观察，轮动早期信号
- **⏸️ 中性板块**：-0.3 < sector_score < +0.3 → 不碰
- **❄️ 回避板块**：sector_score ≤ -0.3 → 不管个股信号多好都过滤

## 5. 第三层：个股K线信号（Stock Signal Layer）

### 5.1 模块位置
- 现有模块基本不变：`patterns.py` / `factors.py` / `strategies.py` / `strategy_portfolio.py`
- 新增入口逻辑在 `src/three_dim_analyzer.py`

### 5.2 关键变化：入口加了过滤

之前：随便输入一个ticker就分析 → 容易在弱势板块里选到"看起来好但大趋势不支持"的票

现在：**只在"强势板块"和"关注板块"内做个股扫描**
1. 从 `company_info.csv` 取出目标板块内的所有成分股
2. 对每只股票加载K线数据
3. 跑现有信号/因子/策略/共识全流程
4. 得到每只股票的 `stock_score ∈ {-1, 0, +1}`

### 5.3 个股评分 `stock_score ∈ {-1, +1}`
直接复用现有多策略共识结果：
- 共识"强烈偏多"(score ≥ 2) → +1
- 共识"偏多"(1 ≤ score < 2) → +0.5
- 共识"中性"(-1 < score < 1) → 0
- 共识"偏空"(-2 < score ≤ -1) → -0.5
- 共识"强烈偏空"(score ≤ -2) → -1

## 6. 三维综合评分体系

### 6.1 最终得分公式

```
final_score = news_score * 0.25     # 消息面权重25%
            + sector_score * 0.35   # 板块权重35%（最重要，赛道对了事半功倍）
            + stock_score * 0.40    # 个股权重40%
```

取值范围：`-3.0 ~ +3.0`

### 6.2 分级建议

| 总分 | 判断 | 操作建议 |
|------|------|----------|
| **+2.0 ~ +3.0** | 🔥 强偏多 | 三维共振，重点关注，可建仓 |
| **+1.0 ~ +2.0** | 📈 偏多 | 两个维度支持，可关注 |
| **-1.0 ~ +1.0** | ⏸️ 中性 | 信号矛盾，观望 |
| **-2.0 ~ -1.0** | 📉 偏空 | 回避 |
| **-3.0 ~ -2.0** | ❄️ 强偏空 | 坚决不碰，板块和消息都不配合 |

### 6.3 硬性过滤规则（一票否决）

- 消息面评分 = -2（强利空）→ 直接过滤，不看个股
- 板块综合评分 ≤ -1（资金确认流出）→ 直接过滤，不看个股
- 这两个条件任一触发，即使个股K线出现最强信号也不推荐

### 6.4 输出格式示例

```
股票: NVDA (英伟达)
板块: 科技 (XLK)

三维评分:
  📰 消息面: +2 (强利好: AI芯片需求超预期，语义相似度0.92)
  📊 板块:  +1.0 (资金确认流入，RS排名1/11，MA多头排列)
  📈 个股:  +1  (多策略共识: 强烈偏多，MACD背离+放量突破)

综合得分: 2*0.25 + 1.0*0.35 + 1*0.40 = +2.85 🔥 强偏多

策略支持:
  - MACD背离: 夏普2.1，胜率68%，未来5日平均收益+2.3%
  - 放量突破: 夏普1.8，胜率62%
  - 均线多头排列: 夏普1.5，胜率75%

建议: 三维共振，可考虑短线做多，止损-4%，止盈+8%
```

### 10.3 答辩亮点
| 亮点 | 解释 |
|------|------|
| **三维分析** | 不是单一K线，而是消息面+板块+个股自上而下组合验证 |
| **资金轮动逻辑** | 用ETF相对强度代替复杂资金流数据，简洁有力，人能理解 |

| **消息面驱动** | 新闻→板块→个股的逻辑链完整可演示，有"Agent感" |
| **AI辅助但不代替决策** | AI解释统计结果，规则负责信号，判断清晰可控 |
| **最差执行回测** | 已有功能，说明策略在最差滑点下仍有效，不画饼 |
| **三档持仓周期** | 短线/中线/长线对应不同止损止盈，符合真实交易 |

## 13. 一句话总结

> 这个项目做的是一个**三维K线研究Agent**：
> **先看消息面定方向，再看板块ETF确认资金流向，最后用个股K线信号精准择时**——
> 每个推荐都有"消息利好+资金流入+个股信号"三重验证，所有结论人能看懂、历史数据能验证。

# FLOW AI Trader · 选股系统架构 (Stock Screener v2)

> 从500只股票里，用三维分析框架层层过滤，最终输出5~10只高胜率标的

## 1. 选股哲学

### 1.1 核心原则

选股不是"找最好的股票"，而是**用多重漏斗层层过滤，把不值得看的票踢出去**。

```
503只SP500成分股
    ↓ L0 股票池准备
~400只（剔除数据不完整的）
    ↓ L1 基础过滤（流动性/市值/价格）
~380只
    ↓ L2 三维硬过滤（消息面/板块趋势一票否决）
~150只（强势/关注板块内）
    ↓ L3 技术初筛（趋势+动量+量能门槛）
~50只
    ↓ L4 策略信号（35信号+20策略共识>=偏多）
~20只
    ↓ L5 回测验证（最差执行下夏普>1、胜率>55%）
~10只
    ↓ L6 综合评分+仓位建议
5~10只最终推荐
```

### 1.2 为什么要做漏斗而不是直接打分

- **效率**：500只全跑回测太慢，先用轻量级指标快速淘汰80%的票
- **避免过拟合**：回测是最重的武器，放在最后用，避免数据挖掘偏差
- **可解释**：每一层都有明确理由，能回答"为什么这只票被/没被选中"
- **容错**：每一层都有冗余，单一指标失灵不会导致全盘皆输

### 1.3 两种选股模式

| 模式 | 触发方式 | 适用场景 |
|------|---------|---------|
| **纯技术面扫描** | 无新闻输入，点击"扫描全市场" | 日常盘前选股，无特定消息时 |
| **三维驱动选股** | 输入新闻/事件后扫描 | 有重大事件时，自上而下找受益股 |

## 2. 六层漏斗架构总览

```
┌────────────────────────────────────────────────────────────┐
│  L0  股票池     503只SP500 → ticker标准化+板块归属标记        │
├────────────────────────────────────────────────────────────┤
│  L1  基础过滤   流动性/市值/价格/异常值 → 剔除不可交易        │
├────────────────────────────────────────────────────────────┤
│  L2  三维硬过滤  消息面利空(-2)或板块弱势(≤-0.3)→一票否决    │
├────────────────────────────────────────────────────────────┤
│  L3  技术初筛   趋势/动量/量能快速检查 → 淘汰弱势股          │
├────────────────────────────────────────────────────────────┤
│  L4  策略信号   35信号检测+20策略共识 → 找有信号的票         │
├────────────────────────────────────────────────────────────┤
│  L5  回测验证   最差执行+持仓周期 → 验证历史胜率真实性       │
├────────────────────────────────────────────────────────────┤
│  L6  综合评分   多目标加权排序+风控仓位 → 最终推荐列表       │
└────────────────────────────────────────────────────────────┘
```

## 3. 各层详细设计

### 3.0 L0 · 股票池准备（503只）

**数据源**：`data/company_info.csv`

**处理内容**：
- 统一ticker格式（BRK.B ↔ BRK-B 点号/横杠双向兼容）
- 映射GICS板块（Technology→XLK等，已在event_sector_mapping.json中）
- 加载业务摘要备用（用于TF-IDF语义匹配）
- 构建ticker→公司名/板块/行业的快速查找表

**输出**：标准化的候选股票列表（list of dict）

---

### 3.1 L1 · 基础过滤（503→~380只）

**目的**：快速剔除"根本不能交易/数据有问题"的票，毫秒级完成

**过滤规则**（全部为硬性条件）：

| 过滤器 | 规则 | 原因 |
|--------|------|------|
| 数据完整性 | 近6个月交易日 >= 100天 | 数据不足无法计算指标/回测 |
| 最低价格 | 最新收盘价 >= $5 | 仙股/低价股波动异常，滑点极大 |
| 流动性 | 20日均成交量 >= 50万股 | 流动性差难以进出 |
| 异常剔除 | 近20日无单日涨跌>50% | 排除刚经历崩盘/暴涨/M&A的异常标的 |
| 废弃剔除 | 排除已退市/被并购的ticker | 数据脏值清理 |

**实现**：只读本地CSV最后120行，不请求yfinance，~0.5秒完成。

---

### 3.2 L2 · 三维硬过滤（~380→~150只）

**目的**：只在"有板块支撑"的赛道里选股，弱势板块一票否决

#### 3.2.1 板块评分

调用 `sector_rotation.analyze_sectors(news_scores)` 得到11个板块的综合评分：

| 板块等级 | sector_score范围 | 处理方式 |
|----------|-----------------|---------|
| 🔥 强势板块 | >= +0.8 | ✅ **全部保留**，优先选股 |
| 👀 关注板块 | +0.3 ~ +0.8 | ✅ 保留（轮动早期信号） |
| ⏸️ 中性板块 | -0.3 ~ +0.3 | ❌ 剔除（除非个股信号极强） |
| ❄️ 回避板块 | <= -0.3 | 🚫 **硬否决**，该板块全部剔除 |

#### 3.2.2 消息面加成（有新闻输入时）

- 板块 `news_score <= -2`（强利空）→ 硬否决
- 板块 `news_score >= +2`（强利好）→ 即使ETF评分只在"关注"档也保留（抢跑轮动早期）
- 无新闻输入时只用技术面板块评分

#### 3.2.3 板块内保留数量

- 强势板块：板块内全部股票保留（约40只/板块）
- 关注板块：板块内股票保留，但L3/L4的门槛提高
- 中性/回避板块：剔除

**预计保留**：2~3个强势板块 + 1~2个关注板块 ≈ 100~180只

---

### 3.3 L3 · 技术面初筛（~150→~50只）

**目的**：用轻量指标快速淘汰技术面弱势股，不跑完整回测

对剩余股票加载6个月日线，计算基础指标：

#### 3.3.1 趋势门槛（必须满足至少2条）
- [ ] 价格在MA60之上（中期趋势向上）
- [ ] MA20 > MA60（中期均线多头）
- [ ] MA20斜率 > 0（短期均线向上）
- [ ] 价格距离20日低点 > 5%（不在创新低）
- [ ] ADX > 20（有趋势，不是纯震荡）

#### 3.3.2 动量大关（必须满足至少1条）
- [ ] RSI(14) 在 40~75 之间（不超买不超卖，有上攻空间）
- [ ] MACD柱 > 0 或 MACD金叉在10日内
- [ ] 近20日涨幅跑赢SPY（相对强度为正）

#### 3.3.3 一票否决（任一触发即淘汰）
- RSI > 80（严重超买，追高风险大）
- 价格跌破MA60超过10%（深度下跌趋势）
- 近10日有单日跌幅>15%（崩盘/利空未消化）
- 布林带极度收窄后向下突破（变盘向下）

#### 3.3.4 加分项（用于排序取Top50）
- 近5日均量 > 20日均量（近期放量）
- 最近1日涨幅>2%且量能>1.5倍均量（放量上涨）
- OBV创新高（资金持续流入）
- 强势板块内（+5分）/ 关注板块内（+2分）

**性能**：每只票只算20个左右基础指标，~150只预计10秒内完成。

---

### 3.4 L4 · 策略信号筛选（~50→~20只）

**目的**：找"现在有明确策略信号触发"的票

对剩余~50只跑完整因子/信号/策略管线：

1. `add_all_factors(df)` — 85个因子全部计算
2. `add_indicators(df)` + `detect_all(df)` — 35个K线信号
3. `get_latest_consensus(df)` — 20策略共识

#### 3.4.1 信号得分计算

| 条件 | 得分 |
|------|------|
| 多策略共识 = 强烈偏多 | +10 |
| 多策略共识 = 偏多 | +6 |
| 近5日看涨信号数 - 看跌信号数 ≥ 3 | +5 |
| 近5日看涨信号数 - 看跌信号数 = 1~2 | +2 |
| 近1日出现经典看涨形态（锤头/晨星/刺透/大阳线突破） | +3 |
| 强势板块内股票 | +4 |
| 多策略共识 = 中性 | 0 |
| 多策略共识 = 偏空 | -8 |
| 多策略共识 = 强烈偏空 | -15（直接淘汰） |

#### 3.4.2 保留条件
- 共识方向非偏空/强烈偏空
- 信号得分 >= 10
- 至少1个策略当前发出BUY/STRONG_BUY
- 至少2个看涨信号在近5日内触发

**输出**：~20只候选股，每只带信号明细和共识评分。

---

### 3.5 L5 · 回测验证（~20→~10只）

**目的**：用最差执行（买入最高、卖出最低）的严格回测排雷。
只相信"滑点拉满还能赚钱"的策略。

#### 3.5.1 回测参数（worst_case=True，最严标准）

| 参数 | 短线(short) | 中线(medium) | 长线(long) |
|------|------------|-------------|-----------|
| 持仓天数 | 1~15天 | 15~30天 | 30~180天 |
| 止损 | -4% | -6% | -10% |
| 止盈 | +8% | +12% | +30% |
| 买入价 | 当日最高价 | 同左 | 同左 |
| 卖出价 | 当日最低价 | 同左 | 同左 |
| 佣金 | 0.1% | 0.1% | 0.1% |
| 最低交易次数 | >= 3次 | >= 3次 | >= 2次 |

#### 3.5.2 硬性门槛（不满足即淘汰）

| 指标 | 最低要求 | 优秀标准 |
|------|---------|---------|
| 最佳策略夏普比率 | >= 0.8 | >= 1.5 |
| Top5策略平均夏普 | >= 0.5 | >= 1.2 |
| 最佳策略胜率 | >= 52% | >= 65% |
| Top5等权组合收益 | > 0%（最差执行下不能亏） | > 10% |
| 最大回撤 | < -15% | < -8% |
| 盈亏比 | >= 1.2 | >= 2.0 |
| 交易次数 | >= 3 | >= 5 |

#### 3.5.3 策略一致性检查
- Top5策略中至少3个方向一致看多 → 保留
- Top5策略方向混乱（2多2空1中性）→ 扣10分
- Top5策略看空方向居多 → 淘汰

**输出**：每只股票的最佳策略、Top5夏普均值、组合收益、胜率、回撤等关键指标。

---

### 3.6 L6 · 综合评分 + 最终推荐（~10→5~10只）

#### 3.6.1 综合评分公式（满分100分）

```
final_score = 
    ── 三维维度（30分）──
    news_score_norm      * 5     # 0~10分  消息面（有新闻时）
    sector_score_norm    * 7.5   # 0~15分  板块强度（最重要）
    consensus_score_norm * 2.5   # 0~5分   个股策略共识方向

    ── 信号维度（25分）──
    signal_bull_count    * 2     # 0~10分  看涨信号数量
    signal_quality       * ...   # 0~15分  信号质量（反转/突破/共振加权）

    ── 回测维度（35分）──
    best_sharpe          * 5     # 0~12分  最佳策略夏普
    avg_sharpe_top5      * 4     # 0~8分   Top5平均夏普
    win_rate             * 10    # 0~10分  胜率
    portfolio_return     * 0.3   # 0~5分   组合收益（最差执行）

    ── 风险维度（10分）──
    max_drawdown_score           # 0~10分  回撤越小分越高
```

#### 3.6.2 推荐评级

| 综合得分 | 评级 | 建议单票仓位 | 建议 |
|---------|------|-------------|------|
| 80~100 | ★★★★★ S级 | 15%~20% | 三维共振，优先建仓 |
| 65~79 | ★★★★ A级 | 10%~15% | 多维度支持，重点关注 |
| 50~64 | ★★★ B级 | 5%~10% | 有亮点但有瑕疵，轻仓试探 |
| < 50 | 不推荐 | 0% | 淘汰 |

#### 3.6.3 动态仓位调整

基础仓位按评级确定，再乘以三个系数：

```
position_pct = base_pct
             * regime_multiplier      # 市场状态系数：强趋势上涨1.2 / 震荡0.8 / 强趋势下跌0.4
             * volatility_multiplier  # 波动率系数：低波(ATR<MA20*0.8)1.2 / 高波(ATR>MA20*1.3)0.7
             * conviction_multiplier  # 三维共振系数：三维全部看多1.3 / 单维度0.8
```

**风控硬约束**：
- 单只股票仓位 ≤ 总资金20%
- 同时持仓 ≤ 8~10只股票（行业分散）
- 单一板块仓位 ≤ 总资金40%（避免板块集中风险）
- 总仓位根据VIX动态调整：VIX<15满仓 / VIX15-25 80% / VIX25-35 50% / VIX>35 30%

#### 3.6.4 输出格式（每只推荐股）

```

  │

  ├─ backtest_verify(stocks, hold)      # L5: ~10只
  │   ├─ backtest_all(df, worst_case=True)
  │   └─ backtest_portfolio(df, top5)
  ├─ score_and_rank(stocks)             # L6: 评分排序
```

## 8. 性能优化

### 8.1 并行计算
- L3~L5的个股分析互相独立，使用 `concurrent.futures.ThreadPoolExecutor` 并行
- 线程数：默认8线程（yfinance IO密集型，线程优于进程）
- 预计扫描500只从原来的10分钟降到~1分钟

### 8.2 数据缓存
- 已计算过的股票指标/回测结果缓存到内存（TTL=5分钟）
- 板块分析结果缓存（TTL=15分钟，盘中不需要每次重算）
- 避免重复请求yfinance（data_loader.py已有缓存机制）

### 8.3 渐进式输出
- API支持SSE流式输出，每层过滤完就推送进度
- 前端实时更新漏斗数字，不用等全部完成才显示
- L5（回测）是瓶颈，L4结果出来就可以先展示候选股

## 9. 风险控制规则（内建）

选股系统不仅仅是找涨的票，还要主动排雷：

| 风控规则 | 处理 |
|---------|------|
| VIX > 35（极度恐慌） | 自动降低总仓位至30%，只选S级票 |
| 单一板块持仓 > 40% | 警告并建议分散 |
| 单只股票仓位 > 20% | 强制降仓 |
| 持股数 > 10只 | 提示减仓集中 |
| 推荐股票数 < 3只 | 提示"当前市场无好机会，建议观望" |
| 强势板块=0（所有板块中性/弱势） | 输出"防御"信号，建议空仓或持有现金 |

## 10. 与三维架构的关系

```
THREE_DIM_ARCHITECTURE.md（三维架构v2）
    ↓ 定义了消息面/板块/个股三层怎么分析
    ↓
SCREENER_ARCHITECTURE.md（本文档）
    ↓ 定义了如何用三层分析能力去批量扫500只票
    ↓
最终输出：5~10只高胜率推荐股 + 仓位建议 + 操作计划
```

- 三维架构回答"怎么分析一只股票"
- 选股架构回答"怎么从500只里找出值得分析的那几只"
- 两者是"战术"和"战略"的关系

## 11. 不做的事

- ❌ 高频/做市策略（只做日K级别）
- ❌ 做空推荐（MVP只做多头，可扩展）
- ❌ 期权/衍生品策略
- ❌ 财报/基本面量化（PE/PB/ROE等基本面因子不在MVP范围）
- ❌ 机器学习选股模型（保持规则引擎可解释）
- ❌ 自动下单（只输出建议，人工决策）

## 12. 一句话总结

> 选股系统像一个漏斗：500只股票从大口进，经过**基础过滤→板块筛选→技术初筛→信号验证→回测排雷→综合排序**六层，只有在"好赛道+好趋势+好信号+历史验证"四个维度都过关的股票才能从小口出来——最后给你的不是一只"可能涨"的票，而是一只"不涨都难"的票。
